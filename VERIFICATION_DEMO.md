# 🔐 验证系统演示

## 🎯 问题解决

你之前提到的问题：
> "有没有办法通过一次答题验证后一段时间内不需要再次验证 另外现在验证通过进入页面后每篇博客的阅读还要再次验证 这非常的不方便"

## ✅ 现在已完全解决！

### 1. 一次验证，长期有效
- **默认**: 24小时内无需重复验证
- **可配置**: 可以设置为永久有效或其他时长
- **跨会话**: 关闭浏览器后重新打开仍然有效

### 2. 进入页面后无需重复验证
- **自动检测**: 系统自动检查验证状态
- **无缝访问**: 验证通过后，可以自由访问所有diary内容
- **智能重定向**: 未验证用户自动重定向到首页

## 🚀 如何使用

### 首次访问
1. 点击导航栏的"🔒 Secret"按钮
2. 回答验证问题（如：英文名、城市、饮料）
3. 验证通过后自动跳转到diary页面

### 后续访问
- **24小时内**: 直接访问任何diary页面，无需验证
- **已过期**: 自动重定向到首页，需要重新验证
- **清除验证**: 点击验证状态栏的"清除验证"按钮

## ⚙️ 配置选项

在 `_data/secret_gate.yml` 中设置：

```yaml
# 验证有效期（小时）
verification_duration: 24  # 24小时
# verification_duration: 0   # 永久有效
# verification_duration: 48  # 48小时
```
